HTML


    Definition


  • HTML elements define the content and the framework of a webpage.

    • Types


  • Semantic HTML includes elements where the name of the element describes the role it plays (ex:header).
  • Non-semantic HTML includes elements where the name does not decribe the role it plays (ex: div).
  • Non-semantic HTML is used when it is the most efficient solution to a structural problem.
  • Semantic HTML is preferred as it makes it easier to understand the code and collaborate.

    • Getting Started


  • HTML elements usually have an opening and closing tag within angle brackets (<>).
  • The closing tag includes a forward slash (/) after the opening angled bracket (<).
  • The content is inserted between the opening and closing tags.
  • HTML code opens with the line !DOCTYPE html inside angled brackets.
  • The !DOCTYPE html declaration tells the browser what document type it should expect when it is opened.
  • Second, we have html lang="en" for webpages in english.

    • The Elements


  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The style element defines the CSS styles associated with a website.
  • The meta element defines metadata such as the character set, description, keywords, author and viewpoint.
  • The link element connects an external HTML document.
  • The base element defines the base URL for a webpage.
  • The script element embeds client-side scripts in the HTML document.
  • The div element is used to create a division in the page.
  • The span element is used to create an inline division.
  • Heading elements h1, h2, h3, h4, h5 and h6 range is size with h1 being most prominent.
  • The p element represents a paragraph or block of text.
  • The ul element represents unordered lists.
  • The ol element represents ordered lists.
  • The li element represents list items.
  • The img element contains information about images that are displayerd on the webpage.
  • The img elements contain various attributes, most cimportantly the src attribute which defines the location of the imagae.
  • The br element crated a line of empty space, or a line break between two blocks of content.
  • The a element stands for anchor, which creates links to the same webpage or other webpages.
  • The opening anchor tag includes a href= followed by your link.
  • The footer element appears at the botom of the page and usually includes the author, copyright, contact, sitemap, and navigation.

CSS


    Definition


  • Cascading Style Sheets (CSS) is used to add style to a webpage.

    • Types


  • Inline CSS is used when a developer wants to style an element directly in the HTML file.
  • Internal CSS Style Sheet embeds an entire style sheet directly into the HTML file, in the style tag element, within the head element.
  • External Style Sheets links a file using a link element placed in the HTML's head element.
  • External Style Sheet is the preferred method, as it makes design changes easier.
  • External stylesheets are linked between the head tags of the html using the link element.

    • Getting Started


  • CSS Rule includes a selector and one or more properties with their values in the declaration block.
  • Selector defines the element or attribute to which the rules, or declarations apply.
  • Declaration includes the property(ex: colour) we want to apply the value to, and the value (ex: blue).
  • Example: Selector { Property: Value; Property: Value;}.
  • CSS rules can apply to more than one selector (ex: header and footer) by adding a comma between the desired selectors.
  • Rather than repeating a rule which would add more lines of code and repetition, two or more elements that will share the same styling can be grouped together. This helps us write Do Not Repeat Yourself (DRY) code.
  • To apply a rule to all of the visible elements on the page we use a * as the selector
  • CSS box model describes every HTML element essentially being a box that consists of margins outside the element, the element's boarders, padding and content.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Assiging colours in CSS can be done semantically (ex: blue, grey...) or by using your desired colours hex #. These can found on this HTML Colour Codes Website


Git


    Definition


  • Git is a version control system that lets you manage and keep track of each version of your project.

    • Getting Started


  • Git is great for tracking changes locally on your own computer, but to collaborate with others, you need to use another tool like GitHub.
  • GitHub is a cloud-based hosting service that allows us to keep track of our projects inside repositories that are stored remotely.
  • Using GitHub not only allows us to store our code, but also to easily collaborate with others.
  • GitHub is connected to Git, which is software on your computer that tracks changes that you make to your code.
  • Your local machine can be connected to your GitHub using your SSH keys.
  • To acess Git, we use the command line or terminal.

    • GitHub Terminology


  • Repository (repo) a folder in which all files and their version are stored.
  • Branch a workspace in which you can make changes that won't affect the live site.
  • Markdown (.md) a way to write in GitHub that converts plain text to GitHub code.
  • Commit Changes a saved record of a change made to a file ithin the repo.
  • Pull Request (PR) the way to ask for changes made to a branch to be merged into another branch that also allows for multiple users to see, discuss and review work being done.
  • Merge after a pull request is approved, the commit will be pulled in (or merged) from one branch to another and then, deployed to the live site.
  • Issues how work is tracked when using git. Issues allow user to reporet new tasks and content fixes, as well as allows users to track progress opn a project board from beginning to end of a specific project.

    • Git Commands


    Set Up


  • git config --global user.name “[first name last name]” set a name that is identifiable for credit when review version history.
  • git config --global user.email “[valid-email]” set an email address that will be associated with each history marker.

    • Set Up and Initalization

  • git init initialize an existing directory as a Git repository.
  • git clone [url] retrieve an entire repository from a hosted location via URL.

    • Stage and Snapshot

  • git status checks what branch we are currently on.
  • git add [file] add a file as it looks now to your next commit (stage).
  • got commit -m "[descriptive message]" commit your staged content as a new commit snapshot.

    • Branch and Merge

  • git branch lists your branches. A * will appear next to the active branch.
  • git branch [branch-name] creates a new branch at the current commit.
  • git checkout switches to another branch and checks it out into the working directory.
  • git checkout -b branch-name creates a new branch and switches to it.
  • git merge [branch] merges the specified branch's history into the current one.

    • Share and Update

  • git push [alias] [branch] transmits local branch commit to the remote repository branch.
  • git pull fetches and merges any commits from the tracking remote branch.

JavaScript


    Definition


  • JavaScript is a programming language that adds interactivity to your website.

    • Getting Started


  • You see your JavaScript in action by connecting the script.js to the index.html.
  • Using the script tags insert the link at the bottom of your HTML under the footer element.

    • Terminology


  • A variable is a named container that allows us to store data in our code.
  • String includes all of the topics that your variable will apply to.
  • An array is a single variable that is used to hold a group of data that is typically related in some way.
  • Index is a unique number that identifies the location of the data item in an array.
  • A for loop uses the predictable pattern of indices to perform a task on all items in an array by allowing a single code block to be executed over and over.
  • Comments are snippets of text that can be added along with code. The browser ignores text marked as comments.
  • Single line comments start with //
  • Multiline comments, or block comments start with /* and end with */
  • Commenting out code makes it inactive without deleting it, just incase you need it later.
  • Functions are a set of instructions that tells a computer how to perform a certain task.
  • Control flow is the order in which a computer executes code in a script.
  • Operators are mathematical symbols that produce a result based on two values (or variables).
  • Conditionals are code structures used to test if an expression returns true or not.
  • True if a statement meets a condition.
  • False of a statement does not meet a condition.
